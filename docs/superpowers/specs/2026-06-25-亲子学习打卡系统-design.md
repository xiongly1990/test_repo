# 亲子学习打卡系统设计文档

**日期：** 2026-06-25
**项目名称：** 亲子学习打卡系统

---

## 1. 项目概述

### 1.1 项目目标

开发一个亲子学习打卡系统，孩子完成学习任务后拍照打卡获取积分和奖励，连续打卡解锁成就树，形成激励闭环。系统包括微信小程序端（孩子独立操作）和Web管理后台（家长操作）。

### 1.2 核心功能

- **拍照打卡**：孩子拍照上传学习成果，家长审核确认
- **连续打卡奖励**：连续打卡天数越多，奖励越丰厚
- **任务树成就**：完成任务解锁成就节点，形成长期目标
- **积分兑换**：积累积分兑换实物奖励（冰淇淋、麦当劳、玩具等）
- **催打卡提醒**：每日19:00提醒未打卡孩子

## 3. 小程序端设计

### 3.1 四大模块

| 模块   | 功能说明                      |
| ---- | ------------------------- |
| 首页   | 今日任务列表，展示待打卡/已完成状态，顶部连击进度 |
| 任务树  | 成就解锁树形结构，逐层点亮，展示进度        |
| 奖励中心 | 积分余额展示，奖励卡片兑换             |
| 我的   | 个人中心，打卡记录，成就，积分统计         |

### 3.2 界面风格

- 活泼的彩色配色，适合儿童独立操作
- 大号任务卡片，清晰的状态展示
- 底部Tab导航

### 3.3 核心页面

**首页 — 今日任务**

- 任务列表：已完成（灰色）、待打卡（彩色按钮）
- 顶部连击进度展示：🔥 连续7天打卡
- 今日任务数统计

**打卡页面**

- 拍照上传界面
- 确认打卡按钮

**任务树页面**

- 树形结构展示成就进度
- 三种状态：已完成（实心圆）、进行中（虚线框+进度条）、未解锁（灰色虚线框）
- 成就节点示例：
  - 第1层：初次打卡
  - 第2层：连续7天（奖励：徽章x1）
  - 第3层：连续30天（进行中）
  - 第4层：连续100天
  - 终极成就：学习达人365天

**奖励中心页面**

- 顶部积分余额：850积分
- 网格布局奖励卡片：冰淇淋(100)、游戏时间(200)、新绘本(300)、毛绒玩具(500)
- 积分不足时按钮变灰

**个人中心页面**

- 头像 + 昵称 + 学习天数
- 三项统计：积分、连击天数、已获成就
- 功能菜单：打卡记录、我的成就、联系家长、设置

## 5. 奖惩机制

### 5.1 奖励规则

- 每次打卡获得积分（根据任务设置）
- 连续打卡解锁成就节点
- 积分累积兑换实物奖励

### 5.2 惩罚规则

| 情况         | 处理方式       |
| ---------- | ---------- |
| 当日19:00未打卡 | 自动推送催打卡提醒  |
| 连续漏打2天     | 连击重置为0     |
| 连续漏打3天     | 扣除已获积分的10% |
| 特殊情况       | 家长可手动补卡    |

## 6. 数据流设计

### 6.1 打卡流程

1. 家长在后台设置学习任务
2. 孩子在小程序查看今日任务
3. 孩子拍照/上传学习成果
4. 家长审核确认
5. 打卡成功，连击+1，获得积分

### 6.2 奖励流程

1. 连续打卡3天 → 解锁小奖励
2. 连续打卡7天 → 解锁中奖励
3. 完成任务树节点 → 领取成就奖励
4. 积分累积 → 兑换实物奖励
5. 家长线下发放奖励

## 7. 技术架构

### 7.1 技术栈

| 层级        | 技术选型                |
| --------- | ------------------- |
| 小程序端      | 微信小程序原生开发           |
| 后台管理前端    | Vue 3 + Element UI  |
| 云端服务 / 后端 | Spring Boot + Redis |
| 数据库       | MySQL               |
| 文件存储      | OSS（打卡照片存储）         |
| 消息推送      | 微信模板消息              |

### 7.2 Redis 应用场景

| 用途   | 说明                      |
| ---- | ----------------------- |
| 用户会话 | 登录 Token 缓存，支持快速鉴权      |
| 连击数据 | 每日打卡状态、连续打卡天数缓存，减少数据库查询 |
| 积分排名 | 实时积分榜单计算                |
| 任务缓存 | 当日任务配置缓存，提升小程序加载速度      |
| 打卡限流 | 防止重复打卡提交                |
